# 需求管控分析自动化工作流 (Demand Control Analysis Workflow)

该技能提供了一套完整的自动化流程，用于从多个数据源导入数据、清洗分析需求风险，并完成多维度的信息补全。

## 核心流程说明

整个工作流共分为 6 个核心步骤：

1.  **导出飞书原始数据**：自动启动浏览器并登录飞书后台，导出最新的需求原始 Excel 数据。
2.  **导出腾讯文档**：自动下载腾讯文档中的《项目跟进表》（用于后续分公司和人员匹配）。
3.  **数据清洗与分类**：对原始数据进行逻辑分析，自动识别“评审超时”、“开发超时”、“上线超期”三类风险需求。
4.  **分公司信息模糊匹配**：将分析结果与《项目跟进表》进行对比，自动识别并填入分公司信息，并将“专区码”插入到结果表第二列。
5.  **关键词映射匹配**：针对未能通过模糊匹配成功的行，使用 `映射表.xlsx` 中的映射关系进行二次匹配。
6.  **运营负责人回填**：根据分公司信息，通过 `省份负责人分配表.xlsx` 自动回填对应的运营负责人。

## 环境要求

- **Python 3.8+**
- **浏览器环境**：需要安装 Chrome 浏览器。
- **关键依赖**：`pandas`, `playwright`, `openpyxl`, `python-calamine`.
- **本地配置文件**：
  - `映射表.xlsx`：包含关键字到专区码的映射。
  - `省份负责人分配表.xlsx`：包含分公司到负责人的映射。

## 执行方式

```powershell
# 执行完整工作流
python workflow.py
```

## 输出产物

- **分析结果文件**：位于 `result` 目录下，文件名为 `需求分析结果_时间戳.xlsx`。
- **文件特点**：已完成自动化的数据清洗、分公司匹配、专区码插入（第二列）及负责人回填。
